# Linux 文件系统

<img src="images/Linux.svg" width="300">

Linux 文件系统可以理解为：**Linux 用来组织和管理文件、目录、设备和权限的一套规则**。

## 1. 基本特点

Linux 采用**树状目录结构**，最顶层是根目录：

```bash
/
```

所有文件、目录、磁盘、U 盘、设备，最终都挂在这个根目录下面。

## 2. 常见目录

| 目录              | 作用               |
| --------------- | ---------------- |
| `/bin`          | 基本命令，如 `ls`、`cp` |
| `/sbin`         | 系统管理命令           |
| `/etc`          | 配置文件             |
| `/home`         | 普通用户家目录          |
| `/root`         | root 用户家目录       |
| `/var`          | 日志、缓存、运行数据       |
| `/tmp`          | 临时文件             |
| `/usr`          | 用户程序和共享资源        |
| `/opt`          | 第三方软件            |
| `/dev`          | 设备文件             |
| `/proc`         | 内核和进程信息          |
| `/mnt`、`/media` | 临时挂载点            |

## 3. 文件系统类型

常见 Linux 文件系统有：

| 类型    | 说明           |
| ----- | ------------ |
| ext4  | 最常见、稳定       |
| XFS   | 适合大文件和服务器    |
| Btrfs | 支持快照、校验等高级功能 |
| tmpfs | 基于内存的临时文件系统  |
| NFS   | 网络文件系统       |

## 4. 挂载的概念

Linux 不像 Windows 那样用 C 盘、D 盘表示磁盘。

Linux 会把磁盘分区挂载到某个目录，例如：

```bash
/dev/sdb1  ->  /data
```

访问 `/data`，其实就是访问这个磁盘分区。

## 5. 常用查看命令

```bash
df -h        # 查看磁盘空间
du -sh *     # 查看目录大小
mount        # 查看挂载情况
lsblk        # 查看磁盘和分区
blkid        # 查看文件系统类型和 UUID
```

## 6. 权限

Linux 文件权限主要分三类：

```bash
r  读
w  写
x  执行
```

对应三类用户：

```bash
所有者  所属组  其他人
```

例如：

```bash
-rwxr-xr--
```

表示：

* 所有者：可读、可写、可执行
* 所属组：可读、可执行
* 其他人：可读

一句话总结：**Linux 文件系统是一棵从 `/` 开始的大树，所有文件、目录和设备都挂在这棵树上。**
